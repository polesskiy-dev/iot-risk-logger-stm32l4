◆ publishEventToSubscribers()

static osStatus_t publishEventToSubscribers ( message_t *  message)
static

Publishes an event to all subscribed actors.

This function traverses the list of actors subscribed to a given event and dispatches the event message to each actor's message queue. If an actor does not have a message queue, the message is processed immediately using the actor's message handler.

Parameters
[in]messagePointer to the message to be published.

Example usage:

message_t msg = { .event = GLOBAL_CMD_INITIALIZE, .payload.value = 0 };
publishEventToSubscribers(&msg);

◆ ACTORS_LOOKUP_SystemRegistry

actor_t* ACTORS_LOOKUP_SystemRegistry[MAX_ACTORS]
extern

Global registry for all actors in the system.

This array holds pointers to the initialized actors in the system. Each entry corresponds to an actor identified by a unique actor ID. Actors must be initialized and added to this registry during system startup.

Example usage:

ACTORS_LOOKUP_SystemRegistry[CRON_ACTOR_ID] = CRON_ActorInit();
Warning
Ensure that each actor is correctly initialized before adding it to the registry.

Event subscription matrix for the system's global events.

This 2D array maps each global event to the list of actors that are subscribed to it. Each row corresponds to a specific global event, and each column contains the IDs of actors that are interested in that event.

Note
The matrix is initialized at compile-time and stored in Flash memory for efficiency.
The matrix includes Actors IDs, not Actors themselves to decouple the event manager from the actors.

Example usage:

EV_MANAGER_SubscribersIdsMatrix[GLOBAL_CMD_INITIALIZE] = {CRON_ACTOR_ID, PWRM_MANAGER_ACTOR_ID, ...};
